# /review

Review the code I've selected or the recent changes for:

- Potential bugs or edge cases
- Security concerns
- Performance issues
- Code style and best practices
- Documentation gaps

Be concise and actionable. Provide specific line numbers and code suggestions where applicable.

## Output Format

Structure your review as follows:

1. **Summary**: One-line overview of code quality
2. **Critical Issues**: Must-fix problems (security, bugs)
3. **Recommendations**: Nice-to-have improvements
4. **Positive Notes**: What the code does well
