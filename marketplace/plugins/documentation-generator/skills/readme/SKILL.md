# README Generation Skill

This skill creates professional README files for projects.

## README Sections

Every good README should include:

### Essential
- Project title and description
- Installation instructions
- Quick start / Usage examples
- License

### Recommended
- Features list
- Requirements/prerequisites
- Configuration options
- Contributing guidelines
- Changelog or version history

### Optional
- Badges (build status, coverage, etc.)
- Screenshots or demos
- FAQ section
- Acknowledgments

## Templates

### Library/Package
```markdown
# Package Name

One-line description.

## Installation
## Usage
## API Reference
## Contributing
## License
```

### Application
```markdown
# Application Name

Description of what the application does.

## Features
## Installation
## Configuration
## Usage
## Deployment
## Contributing
## License
```

### CLI Tool
```markdown
# Tool Name

Description of the tool.

## Installation
## Commands
## Examples
## Configuration
## License
```

## Automation

Can automatically:
- Detect project type and language
- Find existing documentation
- Suggest badges based on CI/CD setup
- Generate installation commands from package files
